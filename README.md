# 갈피 (galpi) — 읽기 전 문맥 브리핑

> 글이 어려운 건 독해력 탓이 아니라, 글이 전제하는 문맥이 내게 없기 때문이다.

어려운 뉴스 기사(특히 국제정세·경제)를 넣으면, 그 글이 독자가 이미 안다고 전제하는
개념과 **이전 사건들의 맥락**을 찾아 채워주는 **읽기 전 브리핑**을 만든다.
뉴스는 연재물이다 — 오늘 기사가 안 읽히는 건 대개 지난 편들을 못 봤기 때문이다.
(판결문·공시 등 다른 문서도 입력 가능하지만, 최적화 대상은 뉴스다.)

**요약과의 결정적 차이**: 요약은 원문을 안 읽게 만들고, 브리핑은 원문을 읽을 수 있게 만든다.
그래서 원문의 결론을 스포일러하지 않는 것이 설계 원칙이다.

## 피봇 기록
- **v0.3**: v0.1~0.2는 "문서를 재미있는 썰로 각색"하는 서비스였다.
  하네스를 2회 개선했으나 재미는 소재 의존적임을 확인, "재미"에서 "문맥 이해"로 피봇.
  상세: [docs/planning.md](docs/planning.md) (구 기획서)
- **v0.4**: 사용자 피드백으로 타겟을 뉴스(국제정세·경제 우선)로 좁힘 —
  뉴스의 빠진 문맥은 '이전 사건의 연쇄'라 브리핑의 가치가 가장 크다.
  이름을 행간 → **갈피**로 변경("갈피를 못 잡겠다"의 갈피이자 책갈피의 갈피),
  브랜드 컬러를 프레스 블루로 전환(형광펜 노랑은 하이라이트 전용 유지).
  다음 단계는 크롬 확장(기사 페이지에서 원클릭 브리핑).

## 실행 방법

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # ANTHROPIC_API_KEY 입력
uvicorn app.main:app --reload
# http://localhost:8000
```

## 아키텍처 — 4단계 문맥 파이프라인

```
[문서] → ① 문서 분석 → ② 문맥 채우기 → ③ 브리핑 작성 → ④ 검증 → [읽기 전 브리핑]
         (전제 찾기)    (개념·배경 설명)   (읽기 경로 포함)     ↑
                                          문제 발견 시 수정 재생성 1회
```

| 단계 | 파일 | 역할 | 모델 |
|---|---|---|---|
| ① 문서 분석 | `app/pipeline/analyze.py` | 글이 설명 없이 전제하는 개념·배경 추출 | 저비용 |
| ② 문맥 채우기 | `app/pipeline/contextualize.py` | 각 개념을 "이 글 기준으로" 설명 (일반지식은 표시) | 고품질 |
| ③ 브리핑 작성 | `app/pipeline/compose.py` | 읽기 전 브리핑 + 원문 읽기 경로. 결론 스포 금지 | 고품질 |
| ④ 검증 | `app/pipeline/verify.py` | 문서 모순·일반지식 위장·스포일러 감시 | 저비용 |

## 원칙
- 이 사건의 구체적 사실은 문서 기준. 일반 지식 설명은 "일반 지식" 배지로 구분 표시.
- 원문의 결론을 대신 말하지 않는다 (원문을 읽을 이유를 남긴다).
- 원문 미저장. 비실명 당사자 신원 추정 금지. 법률·투자 자문 아님.

## 크롬 확장 (extension/)

기사 페이지에서 원클릭으로 브리핑을 띄우는 MV3 사이드패널 확장.
개념 카드의 용어가 기사 본문에 형광펜으로 하이라이트되고, 카드를 클릭하면 해당 위치로 스크롤한다.

```
1. 위 로컬 서버 실행 (uvicorn app.main:app)
2. chrome://extensions → 개발자 모드 ON → "압축해제된 확장 프로그램 로드" → extension/ 선택
3. 기사 페이지에서 갈피 아이콘 클릭 → 사이드패널 → "이 기사 갈피 잡기"
```

확장은 얇은 클라이언트다 — 본문 추출(Mozilla Readability.js + 휴리스틱 폴백)과 렌더링만 하고,
LLM 호출은 전부 서버 경유. API 키는 서버 `.env`에만 존재한다.

브리핑 완료 후 **"광고 없이 읽기"**를 누르면 새 탭에 좌측 브리핑 + 우측 기사 본문(이미지 포함)의
2단 리더 뷰가 열린다. 본문은 `chrome.storage.session`(메모리 전용)으로만 전달되어
디스크·서버 어디에도 저장되지 않고, 이미지는 원본 URL을 참조만 한다.
